## Findings

### Finding 1
- Severity: medium
- Category: bug
- Summary: `extractTypographyHints` and `extractSpacingHints` regex capture groups span newlines, causing extracted values to include content from subsequent lines.
- Evidence: In `helpers.ts`, `extractTypographyHints` uses `([\w\s,-]+)`. Because `\s` matches newlines, a line like `Font family: Inter, system-ui\n- Scale: 12px` captures `Inter, system-ui\n- Scale`. Same issue affects `extractSpacingHints` with `([\d\s,pxrem]+)`.
- Recommendation: Replace multi-line-spanning character classes with line-bound alternatives. For example, `([\w\s,-]+)` → `([^\n:]+)` or `([\w\s,-]+?)(?=\n|$)`. Apply the same fix to `extractSpacingHints`.
- Rule candidate: When writing heuristic text extractors with regex, anchor capture groups to the current line to avoid swallowing downstream content.

### Finding 2
- Severity: low
- Category: bug
- Summary: `extractHexColors` misses 8-digit hex colors (with alpha channel) due to `\b` word-boundary assertion.
- Evidence: The regex `#[0-9a-fA-F]{6}\b` fails on `#2D6BE4FF` because `\b` does not match between two word characters (`4` and `F`). The color is silently skipped during seeding.
- Recommendation: Remove `\b` and use `#[0-9a-fA-F]{6}(?=[0-9a-fA-F]{2}|\b|[^0-9a-fA-F]|$)` to optionally allow 8-digit hex, or add a separate pattern for 8-digit hex codes.
- Rule candidate: —

### Finding 3
- Severity: low
- Category: simplification
- Summary: `extractHexColors` duplicates logic for short and long hex matching.
- Evidence: Two separate `match` calls and two separate `forEach` loops for `{3}` and `{6}` hex codes.
- Recommendation: Consolidate into a single pass with `text.match(/#[0-9a-fA-F]{3,8}\b/g)` (after fixing Finding 2), then filter valid lengths in one loop.
- Rule candidate: —

### Finding 4
- Severity: medium
- Category: test-gap
- Summary: No automated tests exist for `isUiStory`, `hasUiTypeOverride`, or `seedDesignFromIntake`.
- Evidence: Verification was performed manually with ad-hoc Node scripts (`/tmp/test-*.js`). The repo has no test harness for extension helpers.
- Recommendation: Add a minimal Node-based test file (e.g., `.pi/extensions/vazir-context/helpers.test.js`) that asserts `isUiStory`/`hasUiTypeOverride` behavior against representative story files, and that `seedDesignFromIntake` produces expected stub content from a mock intake directory.
- Rule candidate: Heuristic extractors and scope-detection logic should have automated regression tests because they are easy to break with small regex changes.

### Finding 5
- Severity: low
- Category: bug
- Summary: `extractTypographyHints` can misinterpret generic "font:" lines as font-family declarations.
- Evidence: A line like `Font: 16px` matches the `font` alternative in the regex and extracts `16px` as the font family.
- Recommendation: Either remove the bare `font` alternative (relying on `font-family` and `typeface` only), or add a negative lookahead to exclude numeric-only values.
- Rule candidate: —
