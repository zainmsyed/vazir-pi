## Findings

### Finding 1
- Severity: medium
- Category: bug
- Summary: `buildMiniConsolidateInstruction` still emits a literal `${path.basename(reviewFilePath)}` placeholder instead of the actual review filename.
- Evidence: In `.pi/extensions/vazir-context/helpers.ts`, `buildMiniConsolidateInstruction(...)` builds the review-file line with a plain quoted string: `"- Read the review file at .context/reviews/${path.basename(reviewFilePath)}."`. Because it is not a template literal, any mini-consolidate prompt produced with a review file will contain the raw `${...}` text rather than a concrete path.
- Recommendation: Change that line to a template literal and add a regression assertion that the rendered instruction contains the real review basename when `reviewFilePath` is provided.
- Rule candidate: When prompt builders interpolate file paths, add one regression assertion that the final prompt contains the concrete rendered path rather than a raw `${...}` placeholder.

### Finding 2
- Severity: medium
- Category: test-gap
- Summary: The new `[fallow]` signal-source validation only proves instruction wording, not that a real downstream consumer actually uses complaints-log signal.
- Evidence: `scripts/validate-vazir-fallow-signal-sources.mts` only asserts that the built instruction strings include `[fallow]`, `.context/complaints-log.md`, and the story label. It does not seed `.context/complaints-log.md` with a `[fallow]` entry and exercise a deterministic consumer path such as candidate generation, learned-rule closeout parsing, or another helper-level flow that proves the new signal can affect behavior beyond prompt text.
- Recommendation: Add one higher-level validation that seeds a story-scoped `[fallow]` complaints-log entry and proves a real consolidate or closeout consumer reads or is influenced by it, rather than only checking string inclusion.
- Rule candidate: When adding a new signal source to an instruction-driven workflow, validate at least one real consumer path end-to-end instead of only checking that the prompt mentions the source.
