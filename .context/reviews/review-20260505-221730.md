## Findings

### Finding 1
- Severity: medium
- Category: bug
- Summary: `extractSpacingHints()` no longer extracts a common spacing-section `Scale:` line.
- Evidence: The current matcher is `/(?:spacing[-\s]?scale)\s*[:=]\s*([^\n]+)/i`, which only matches labels like `Spacing scale:`. A common intake format is `## Spacing` followed by `- Scale: 4, 8, 12, 16`, which story-001's own verification/summary used. That format now leaves `spacingScale` undefined even though the section is clearly spacing-specific.
- Recommendation: Scope extraction to the `## Spacing` section before matching `Scale:`, or prefer `Spacing scale:` globally and fall back to `Scale:` only inside a spacing section. Add a regression assertion for `## Spacing\n- Scale: ...`.
- Rule candidate: When fixing ambiguous regex extraction, prefer section-scoped parsing over deleting useful fallback labels.

### Finding 2
- Severity: medium
- Category: test-gap
- Summary: `helpers.design.test.cjs` is not wired into the standard validation command.
- Evidence: `package.json` runs `node --experimental-strip-types ./scripts/run-validations.mts`, and `scripts/run-validations.mts` only executes files listed in its `validations` array. `.pi/extensions/vazir-context/helpers.design.test.cjs` is outside that list, so `npm test` does not run the new regression tests.
- Recommendation: Add a validation script under `scripts/` that invokes `node .pi/extensions/vazir-context/helpers.design.test.cjs`, and include it in `scripts/run-validations.mts`, or update `npm test` to run the helper test directly.
- Rule candidate: New regression tests must be wired into the project's normal validation command before being considered complete.

### Finding 3
- Severity: low
- Category: test-gap
- Summary: `helpers.design.test.cjs` copies helper implementations instead of exercising the production functions.
- Evidence: The test file contains copied implementations of `extractHexColors`, `extractTypographyHints`, `extractSpacingHints`, `isUiStory`, `hasUiTypeOverride`, and a simplified `seedDesignFromIntake`. If `helpers.ts` changes later but the copied test functions are not updated, tests can pass while production behavior regresses.
- Recommendation: Prefer importing the production module under Node's TypeScript stripping, or extract pure helper logic into exported functions that tests can call directly. If private helpers remain copied, add a clear guard or source-sync check to prevent silent drift.
