## Findings

### Finding 1
- Severity: medium
- Category: bug
- Summary: The Fossil Fallow bridge (`runFallowAuditViaFossilBridge`) calls `git` and `tar` binaries without checking availability first, so a system with Fossil but without Git produces a cryptic bridge-failure warning instead of a clean fallback.
- Evidence: `.pi/extensions/vazir-context/index.ts` lines 436â€“439 call `git init`, `git add`, `git commit`; line 424 calls `tar xzf`. None of these are guarded by `commandExists` or equivalent. If either binary is missing, `execFileSync` throws, the outer `runFallowAudit` catch block shows `Fallow audit bridge failed â€” running LLM-only review`, and the user has no indication that the missing tool was Git or tar rather than Fallow itself.
- Recommendation: Add upfront guards in the Fossil branch of `runFallowAudit` (or inside the bridge) that check `commandExists("git")` and `commandExists("tar")`. If either is missing, return `fallowNotRun("git or tar unavailable for fossil bridge")` with a clear notification so the user understands why Fallow cannot run in Fossil mode on this system.
- Rule candidate: When a feature relies on auxiliary binaries beyond its primary tool, validate each dependency explicitly and emit a diagnostic that names the missing tool so users do not confuse it with a failure of the primary feature.
