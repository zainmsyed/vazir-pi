# Story 015: VCS preference discoverability and override

**Status:** not-started  
**Created:** 2026-05-14
**Last accessed:** —  
**Completed:** —

---

## Goal
Make the `vcs_preference` setting discoverable, documented, and changeable without hand-editing JSON. Currently `vcs_preference` in `.context/settings/project.json` is the only way to force a specific VCS when multiple are present, but no command writes it and no documentation mentions it. Auto-mode also silently prioritises Fossil over Git, which can surprise users who expected Git tooling.

## Verification
A user with both `.git` and `.fslckout` in the same project can run `/setting vcs git` (or a `/vazir-init` reconfigure flow) and the footer immediately switches to Git mode. The `project.json` file is updated and the preference persists across sessions. The setting is documented in a `.context/settings/README.md` file generated by `/vazir-init`.
